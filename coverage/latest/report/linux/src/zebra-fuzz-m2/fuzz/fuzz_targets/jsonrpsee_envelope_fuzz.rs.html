4
//!
5
//! Sister harness to `rpc_handler_fuzz` (Layer-3 trait dispatch). This
6
//! one feeds raw JSON-RPC byte strings into
7
//! `jsonrpsee::RpcModule::raw_json_request(json_str, 1)` so the fuzzer
8
//! exercises the **envelope** layer:
9
//!
10
//!   * `serde_json::from_str::<Request>` — the envelope decoder. This
11
//!     parses `{"jsonrpc":"2.0","method":"...","params":[...],"id":N}`
12
//!     into a `jsonrpsee_types::Request`. Attacker-supplied JSON bombs
13
//!     (deeply-nested arrays, oversized strings, malformed UTF-8) hit
14
//!     this surface first.
15
//!   * `Methods::method(method_name)` lookup → MethodNotFound short-
16
//!     circuit, OR dispatch into `MethodCallback::Async` with raw
17
//!     `RawValue` params.
18
//!   * `Params::sequence()` / `Params::parse::<T>()` — the per-method
19
//!     param decoder generated by `#[rpc(server)]`. This is where
20
//!     `serde_json` deser of every method's parameter tuple happens
21
//!     (e.g. `(String, Option<u8>)` for `getblock`). Type confusion,
22
//!     wrong arity, integer overflow on i32 indices, etc. surface here
23
//!     **before** the handler body runs — the surface is *upstream* of
24
//!     what `rpc_handler_fuzz` covers.
25
//!
26
//! This harness intentionally drives `raw_json_request` with two input
27
//! flavours from the libfuzzer-supplied bytes:
28
//!
29
//!   1. **Structured envelope** — a small leading byte selects a
30
//!      method name from the production list, then we synthesize a
31
//!      JSON-RPC envelope `{"jsonrpc":"2.0","method":"<m>","params":<P>,"id":1}`
32
//!      where `<P>` is built from the remaining bytes (treated as raw
33
//!      JSON if it parses, or as a JSON string literal if not). This
34
//!      drives method-specific `Params::parse` paths.
35
//!   2. **Raw JSON** — when the leading byte selects "raw", the entire
36
//!      remaining buffer is handed to `raw_json_request` verbatim.
37
//!      libFuzzer can then mutate towards arbitrary attacker JSON,
38
//!      including malformed envelopes, missing fields, oversized
39
//!      params, batch flooding (jsonrpsee's HTTP layer handles
40
//!      batches; the `Methods::raw_json_request` API is single-call,
41
//!      so batches are tested at the HTTP layer in production but
42
//!      not here — that requires `Server::start` which we deliberately
43
//!      avoid to keep iterations cheap).
44

46
